# Модуль prospekt.propmodificator — Модификатор свойств ТП (Аспро: Премьер)

## Описание

Модуль для 1С-Битрикс, разработанный для шаблона **Аспро: Премьер**.  
Добавляет на карточку товара возможность ввода **произвольных значений** формата изделия (ширина × высота) и тиража — вместо ограниченного набора кнопок-пресетов.

### Что делает модуль

- Добавляет поля с кнопками ± рядом со стандартными кнопками выбора свойств ТП (формат, тираж)
- Стандартные кнопки остаются видимыми и работают как **пресеты**
- При вводе значения, совпадающего с пресетом, автоматически выбирает соответствующее ТП
- При вводе произвольного значения — выбирает ближайшее ТП и **интерполирует цену** (клиентски, только для отображения)
- При добавлении в корзину **пересчитывает цену на сервере** (клиентская цена игнорируется)
- Записывает кастомные свойства позиции корзины: «Формат (Ш×В)» и «Тираж»

---

## Требования

- 1С-Битрикс любой редакции с модулями **iblock**, **catalog**, **sale**
- Шаблон **Аспро: Премьер** (модуль aspro.premier)
- PHP 8.0+

## Установка

### 1. Загрузите модуль

Скопируйте папку `prospekt.propmodificator` в директорию:

```
/local/modules/prospekt.propmodificator/
```

### 2. Зарегистрируйте модуль в Битрикс

Перейдите в **Административная панель → Marketplace → Установленные решения** (или `/bitrix/admin/partner_modules.php`) и установите модуль **Модификатор свойств ТП**.

При установке модуль автоматически:
- Определит ID инфоблоков товаров и ТП
- Создаст свойства `SET_FORMAT` и `SET_VOLUME` в инфоблоке товаров
- Скопирует JS/CSS в `/bitrix/js/prospekt.propmodificator/`

### 3. Добавьте в init.php

Скопируйте содержимое файла `install/php_interface/init.php.fragment` в:

```
/bitrix/php_interface/init.php
```

### 4. Подключите в шаблоне

В файле `footer.php` вашего шаблона добавьте перед `</body>`:

```php
// Только на страницах детального просмотра товара
if ($APPLICATION->GetCurPage(false) /* проверьте условие для детальной страницы */) {
    include_once $_SERVER['DOCUMENT_ROOT'] . '/local/modules/prospekt.propmodificator/template_include.php';
}
```

Или добавьте прямо в шаблон детальной страницы товара (например, `catalog.element/template.php`):

```php
include_once $_SERVER['DOCUMENT_ROOT'] . '/local/modules/prospekt.propmodificator/template_include.php';
```

## Настройки модуля

Откройте: **Администрирование → Настройки → Настройки модулей → Модификатор свойств ТП**

| Параметр | Описание | По умолчанию |
|---|---|---|
| ID инфоблока ТП | Инфоблок торговых предложений | 15 |
| ID инфоблока товаров | Родительский инфоблок | 14 |
| Код свойства ФОРМАТ | Свойство ТП с форматами (XML_ID вида `210x297`) | `CALC_PROP_FORMAT` |
| Код свойства ТИРАЖ | Свойство ТП с тиражами (XML_ID — число) | `CALC_PROP_VOLUME` |
| Код свойства настроек формата | Multiple-string свойство товара | `SET_FORMAT` |
| Код свойства настроек тиража | Multiple-string свойство товара | `SET_VOLUME` |
| ID типа цены | Тип цены для интерполяции | 1 |

## Заполнение свойств товара

После установки модуля в инфоблоке товаров появятся два свойства.  
Заполните их в редакторе товара:

### SET_FORMAT — Настройки формата

| Значение | Описание |
|---|---|
| `MIN_WIDTH` | `10` |
| `MAX_WIDTH` | `1200` |
| `MIN_HEIGHT` | `10` |
| `MAX_HEIGHT` | `1200` |
| `STEP` | `1` |

Это ограничивает диапазон ввода ширины и высоты (в мм) в форме на сайте.

### SET_VOLUME — Настройки тиража

| Значение | Описание |
|---|---|
| `MIN` | `10` |
| `MAX` | `100000` |
| `STEP` | `1` |

## Требования к свойствам ТП

Для корректной работы интерполяции свойства инфоблока ТП должны иметь правильные **XML_ID** в значениях перечисления:

### CALC_PROP_FORMAT

Каждое значение перечисления должно иметь XML_ID в формате `ШxВ`, например:

- `100x148` — формат А6
- `148x210` — формат А5
- `210x297` — формат А4
- `297x420` — формат А3

### CALC_PROP_VOLUME

Каждое значение перечисления должно иметь XML_ID — целое число:

- `100`
- `500`
- `1000`
- `5000`
- `10000`

## Как работает интерполяция цены

### Только формат изменён (тираж — стандартный пресет)

Линейная интерполяция по площади листа (ширина × высота):

```
area = width × height
price = priceLow + (priceHigh - priceLow) × (area - areaLow) / (areaHigh - areaLow)
```

### Только тираж изменён (формат — стандартный пресет)

Линейная интерполяция по тиражу:

```
price = priceLow + (priceHigh - priceLow) × (volume - volumeLow) / (volumeHigh - volumeLow)
```

### Оба параметра произвольные

Билинейная интерполяция по 4 угловым ТП (ближайшие по площади × ближайшие по тиражу):

```
r1 = lerp(Q11, Q21, areaLow, areaHigh, area)
r2 = lerp(Q12, Q22, areaLow, areaHigh, area)
price = lerp(r1, r2, volumeLow, volumeHigh, volume)
```

Клиентская цена — только для **предварительного отображения**.  
Финальная цена всегда рассчитывается **на сервере** (PHP, класс `PriceInterpolator`).

## Устранение неполадок

### Поля ввода не появляются на странице

1. Проверьте, что `template_include.php` подключён на детальной странице товара
2. Проверьте в консоли браузера: `window.pmodConfig` — должен быть объект с данными
3. Убедитесь, что у товара заполнены свойства `SET_FORMAT` или `SET_VOLUME`
4. Проверьте правильность ID инфоблоков в настройках модуля

### Цена не считается

1. Проверьте XML_ID значений перечисления в свойствах `CALC_PROP_FORMAT` и `CALC_PROP_VOLUME`
2. XML_ID формата должен быть вида `210x297` (цифры × цифры, латинская x)
3. XML_ID тиража должен быть числом: `1000`
4. Проверьте ID типа цены в настройках модуля (должен совпадать с реальным типом)

### При добавлении в корзину цена неправильная

1. Проверьте логи PHP на ошибки класса `BasketHandler`
2. Убедитесь, что обработчики событий зарегистрированы (см. init.php.fragment)
3. Проверьте, что свойства `SET_FORMAT`/`SET_VOLUME` правильно заполнены в редакторе товара
