# aspro-propertys-modificator

## Модификатор свойств торговых предложений (Аспро: Премьер)

**Версия:** 1.2.0
**Разработчик:** [PROSPEKT-WEB](https://prospektweb.ru)
**Идентификатор модуля:** `prospektweb.propmodificator`

### Описание

Модуль для 1С-Битрикс, добавляющий на карточку товара слой кастомизации SKU-свойств с произвольным вводом значений и автоматическим пересчётом цены по существующим торговым предложениям. По умолчанию в комплекте идут сценарии для **формата** (ширина × высота) и **тиража**, но архитектура модуля ориентирована на расширение под другие свойства через конфигурацию и привязки.

### Решаемые задачи

- **Произвольный ввод кастомизируемых SKU-свойств** — покупатель может вводить значения, не ограничиваясь фиксированными пресетами ТП
- **Автоматический расчёт цены** — цена вычисляется по существующим ТП с интерполяцией (линейной/билинейной, в зависимости от конфигурации и доступных измерений)
- **Стабильный lifecycle UI** — единая оркестрация обновления заголовка и цены после стабилизации SKU (без моргания и рассинхрона)
- **Серверная валидация и финальный пересчёт** — защита от некорректных значений и подмены клиентской цены
- **Запись параметров в корзину** — сохранение фактических пользовательских параметров и финальной расчётной цены

### Возможности для пользователей

- Настройка кастомизируемых SKU-свойств и их диапазонов на уровне товара (стандартные профили: `SET_FORMAT` и `SET_VOLUME`)
- Конфигурация привязки к конкретным инфоблокам товаров и торговых предложений
- Выбор типа цены для интерполяции
- Настройка символьных кодов используемых SKU-свойств через административную панель
- Автоматическое определение инфоблоков каталога Аспро: Премьер при установке
- Автоматическое создание базовых служебных свойств (`SET_FORMAT`, `SET_VOLUME`) в инфоблоке товаров
- Валидация SKU-свойств ТП (`CALC_PROP_FORMAT`, `CALC_PROP_VOLUME`) при установке с выводом предупреждений
- **Режим отладки** — запись в лог причин неактивации для диагностики

### Требования

#### Платформа
- **1С-Битрикс**: редакция «Малый бизнес» или выше (необходимы модули `iblock`, `catalog`, `sale`)
- **Шаблон**: Аспро: Премьер (модуль адаптирован для интеграции в шаблон через `template_include.php`)

#### PHP
- **PHP 8.0+** (используются union types `int|float`, named arguments, typed properties)

#### Обязательные модули 1С-Битрикс
- `iblock` — Информационные блоки
- `catalog` — Торговый каталог
- `sale` — Интернет-магазин

### Особенности

- **Открытый код** — весь код модуля открыт, закодированных (Zend/ionCube) файлов нет
- **Нет внешних зависимостей** — модуль не требует подключения или оплаты сторонних сервисов и API
- **Клиент-серверная архитектура** — калькулятор цены работает и на клиенте (JS), и на сервере (PHP) с идентичной логикой интерполяции
- **Кэширование конфигурации** — класс `Config` кэширует настройки в памяти для минимизации запросов к базе данных
- **Автоподключение без правки шаблона** — установщик создаёт файл в `/bitrix/php_interface/include/`, поэтому модуль работает даже при отсутствии `init.php`
- **Поддержка URL с offer-параметром** — корректно определяет товар при переходе по URL вида `?oid=3401` (Аспро: Премьер)

### Установка (рекомендованный путь — `/local/modules`)

> Файлы модуля рекомендуется размещать в `/local/modules/prospektweb.propmodificator/`.
> Это стандартное место для кастомного кода: он не затрагивается обновлением ядра Битрикс.

1. Скопировать содержимое репозитория в `/local/modules/prospektweb.propmodificator/`
2. Перейти в административную панель: **Marketplace → Установленные решения**
3. Найти модуль «Модификатор свойств ТП (Аспро: Премьер)» и нажать **Установить**
4. В мастере установки указать (или подтвердить автоопределённые) ID инфоблоков товаров и торговых предложений
5. При установке инсталлятор:
   - регистрирует обработчик события `main::OnEpilog` в базе данных;
   - создаёт файл `/bitrix/php_interface/include/prospektweb_propmodificator.php`
     для гарантированного подключения даже без `init.php`.
   
   **Правки шаблона Аспро не требуются.**
6. После установки на детальной странице товара в `<head>` появятся ассеты
   `/bitrix/js/prospektweb.propmodificator/script.js`, `style.css` и `window.pmodConfig`.

#### Перенос из `/bitrix/modules` в `/local/modules`

Если модуль уже установлен в `/bitrix/modules/prospektweb.propmodificator/`:

1. Деинсталлируйте модуль через **Marketplace → Установленные решения** (файлы включат удаление include-файла).
2. Скопируйте папку:
   ```
   cp -r /bitrix/modules/prospektweb.propmodificator /local/modules/prospektweb.propmodificator
   ```
   или загрузите свежие файлы из GitHub в `/local/modules/prospektweb.propmodificator/`.
3. Удалите старую копию из `/bitrix/modules/` (опционально, но рекомендуется):
   ```
   rm -rf /bitrix/modules/prospektweb.propmodificator
   ```
4. Установите модуль заново через административную панель.

### Альтернативное подключение (без установщика)

Если нужно подключить модуль вручную, добавьте следующий код в
`/local/php_interface/init.php` или `/bitrix/php_interface/init.php`:

```php
\Bitrix\Main\EventManager::getInstance()->addEventHandler(
    'main',
    'OnEpilog',
    function () {
        if (!\Bitrix\Main\Loader::includeModule('prospektweb.propmodificator')) {
            return;
        }
        $f = \Bitrix\Main\Application::getDocumentRoot()
            . getLocalPath('modules/prospektweb.propmodificator/template_include.php');
        if (file_exists($f)) {
            include_once $f;
        }
    }
);
```

### Настройка

Страница настроек: `/bitrix/admin/settings.php?mid=prospektweb.propmodificator`

| Параметр | По умолчанию | Описание |
|---|---|---|
| ID инфоблока ТП | `15` | Инфоблок, где хранятся торговые предложения |
| ID инфоблока товаров | `14` | Родительский инфоблок товаров |
| Код свойства ФОРМАТ в ТП | `CALC_PROP_FORMAT` | Свойство типа «Список» с XML_ID вида `210x297` |
| Код свойства ТИРАЖ в ТП | `CALC_PROP_VOLUME` | Свойство типа «Список» с числовым XML_ID (`1000`) |
| Код свойства настроек формата | `SET_FORMAT` | Множественное строковое свойство товара |
| Код свойства настроек тиража | `SET_VOLUME` | Множественное строковое свойство товара |
| ID типа цены | `1` | Тип цены из справочника (базовая) |
| Фильтр URL | `/catalog/` | Подстрока URL для активации (пусто = все страницы) |
| Режим отладки | `N` | Запись в `/bitrix/logs/prospektweb.propmodificator.log` |

### Подготовка данных в свойствах товара

**SET_FORMAT** (множественное, с описанием):

| Значение (ключ) | Описание (значение) |
|---|---|
| `MIN_WIDTH` | Минимальная ширина, мм |
| `MAX_WIDTH` | Максимальная ширина, мм |
| `MIN_HEIGHT` | Минимальная высота, мм |
| `MAX_HEIGHT` | Максимальная высота, мм |
| `STEP` | Шаг изменения, мм |

**SET_VOLUME** (множественное, с описанием):

| Значение (ключ) | Описание (значение) |
|---|---|
| `MIN` | Минимальный тираж, шт. |
| `MAX` | Максимальный тираж, шт. |
| `STEP` | Шаг изменения, шт. |

### Диагностика

Если модуль не проявляется на сайте:

1. Включите «Режим отладки» в настройках модуля.
2. Откройте детальную страницу товара (URL вида `/catalog/.../?oid=3401`).
3. Проверьте лог: `/bitrix/logs/prospektweb.propmodificator.log`.
4. Лог покажет: фильтр URL, найденный `productId`, наличие `SET_FORMAT`/`SET_VOLUME`.

Также полезна диагностика в консоли браузера:

```js
(() => {
    const has = n => document.head.querySelectorAll(`[src*="${n}"],[href*="${n}"]`).length;
    console.table({
        hasPmodConfig: !!window.pmodConfig,
        pmodScriptInHead: has('prospektweb.propmodificator/script'),
        pmodStyleInHead:  has('prospektweb.propmodificator/style'),
    });
})();
```
